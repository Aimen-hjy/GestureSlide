# GestureSlide 项目审查报告

审查范围：所有顶层 Python 文件、依赖、README、模型文件、24 个训练数据文件及项目打包结构。

## 已在修订版中处理的确定性问题

1. 修复真实数据缺少 `NONE` 时 `classification_report` 的 10/11 类数量冲突。
2. 修复 `predict_proba` 列下标被误当成真实类别 ID 的问题；现在使用 `model.classes_`。
3. 修复规则回退中的不存在枚举 `PINKY_ONLY`、`PEACE`。
4. 修复 `classify(None)` 调用不存在的 `_reset_swipe()`。
5. 修复小指状态检测错误使用 DIP 而非 PIP，减少合成训练/运行时特征不一致。
6. 模型存在但 scaler 缺失或维度不匹配时不再静默启用错误推理。
7. 鼠标移动改为每帧连续处理，不再每 5 帧才移动一次。
8. 模型路径改为相对代码目录，不再依赖启动时的当前工作目录。
9. 训练数据加载增加特征长度、NaN/Inf、标签 ID 和标签名称一致性检查。
10. requirements 移除重复的 OpenCV 包，并显式加入 joblib。
11. 更新 `.gitignore`，清理 `.venv`、`__pycache__` 等非项目文件。
12. README 更正最低 Python 版本和 Debian/Ubuntu 虚拟环境安装方法。

## 数据审查结果

共 3363 条、24 个文件、10 个实际类别；没有 `NONE` 负样本。

| 类别 | 样本数 |
|---|---:|
| OPEN_PALM | 496 |
| FIST | 617 |
| LEFT_POINT | 116 |
| RIGHT_POINT | 126 |
| DOWN_POINT | 480 |
| POINT_INDEX | 278 |
| THUMB_UP | 308 |
| PEACE_UP | 369 |
| PEACE_DOWN | 190 |
| THREE_FINGERS | 383 |

原来的随机帧拆分准确率为 98.66%。这会把同一段视频的相邻帧同时放进训练集和测试集。
作为诊断，我按采集文件留出 8 个有重复采集的类别进行测试，准确率约为 27.4%。
这个数字不是最终模型性能结论，但说明当前 98.66% 存在严重会话泄漏，不能用于说明实机泛化能力。

建议每类至少采集 3–5 个独立会话，改变距离、角度、光照、背景和手部位置；
LEFT_POINT、RIGHT_POINT 当前各只有一个采集文件，无法做真正的会话级测试。

## 尚需进一步设计的重点

### 1. 必须采集 `NONE` 负样本

模型仅在检测到手时运行，因此 `NONE` 应代表“检测到手，但姿势不是任何命令”。
没有负样本时，未知姿势会被强制归入某个命令类，可能误翻页、退出或点击。

### 2. 训练/测试应按采集文件分组

真实数据应记录 `session_id`，使用 `GroupShuffleSplit` 或 `GroupKFold`。
混合训练时，测试集只使用独立的真实会话；合成样本不能进入最终测试集。

### 3. 特征需要按手掌尺寸归一化

当前只减去手腕坐标，没有做逐样本尺度归一化；模型容易学习摄像头距离和个人手型。
可用腕点到中指 MCP 的距离或食指 MCP 到小指 MCP 的距离作为尺度，并同时归一化捏合距离。

### 4. 捏合检测容易误点击

当前只依据固定阈值和三根手指状态。拳头或收拢姿势也可能满足条件。
建议使用相对手掌尺寸的捏合距离，并只在“未捏合 → 捏合”的边沿触发一次点击。

### 5. 键盘事件只发送到当前焦点窗口

`pyautogui` 不会自动找到 PowerPoint/Impress。预览窗口处于焦点时，F5、左右键和 Esc
可能发送给预览窗口。应增加平台控制后端：

- Windows：PowerPoint COM 或明确激活 PowerPoint 窗口；
- Linux/X11：LibreOffice UNO、xdotool/wmctrl；
- Wayland：优先使用应用 API，而不是全局键鼠模拟。

### 6. 鼠标建议改为绝对坐标映射

当前相对位移方案会漂移并产生惯性。建议将摄像头内一个 ROI 映射到整块屏幕，
使用 `moveTo` 和低通滤波处理绝对目标坐标。

### 7. 手指状态算法仍需改进

拇指伸展仅比较 x 差值，对竖拇指、左右手和手掌旋转不稳定。
当前数据中 THUMB_UP 的多数样本甚至被手指状态算法标成拇指未伸展。
建议改用关节夹角和骨段方向，或减少这些人工布尔特征对模型的权重。

### 8. 模型文件应包含元数据

应将模型、scaler、标签表、特征版本、sklearn 版本打包为一个文件。
特征或标签发生变化时，旧模型应明确拒绝加载，而不是静默产生错误结果。

### 9. 部分功能和配置未接入

- `DOWN_POINT`、`FIST`、`THREE_FINGERS` 已训练但没有动作。
- `toggle_black_screen`、`toggle_white_screen`、`pause_resume` 等方法没有调用。
- `SWIPE_THRESHOLD`、`SWIPE_HISTORY_SIZE`、`SWIPE_COOLDOWN` 未使用。
- `TWO_FINGERS_CLOSE` 枚举未使用。

应删除死代码，或明确设计手势到动作的映射。

### 10. 增加自动化测试

至少覆盖：

- 模型类别缺少 `NONE` 时的 ID 映射；
- 模型/scaler 缺失与维度不匹配；
- 规则回退所有手指组合不崩溃；
- 训练数据格式校验；
- 手势稳定器和冷却逻辑；
- 控制器动作路由（使用 mock，不能真的操作键鼠）。

## 打包问题

原压缩包约 364 MB，解压后约 1.13 GB，其中 `.venv` 约 1.12 GB。
虚拟环境不可跨机器可靠复用，也可能因 ZIP 对符号链接的处理而损坏。
分发包不应包含 `.venv`、`.git` 或 `__pycache__`；只保留源码、requirements、
必要模型和可选训练数据。
